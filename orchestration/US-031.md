# US-031: Enhance user profile page with identity cluster and timeline

**Priority:** 31
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the user profile page shows a complete identity cluster (resolved user_id + all linked device_ids), a polished event timeline with source identity labels and merged-event indicators, and lazy loading for large event histories.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- User lookup page (from Tier 1) is enhanced with an identity cluster section at the top
- Identity cluster shows: resolved user_id (primary), all linked device_ids listed as secondary identifiers with a Device lucide-react icon, first_seen and last_seen timestamps in human-readable format
- Event timeline below the cluster shows all events attributed to this user in chronological order (oldest first)
- Each event in the timeline shows: timestamp, event name, source identity (which device_id or user_id sent it, with a label like 'via device-abc'), all properties as key-value pairs
- Events that were originally anonymous (sent with device_id only, later merged) have a subtle visual indicator (e.g., a small 'merged' badge or different background shade)
- Timeline supports lazy loading or pagination: initially loads the latest 100 events, 'Load more' button or infinite scroll to fetch earlier events
- GET /api/users/:id endpoint is enhanced to also return the identity cluster and event count
- Typecheck passes
- Verify in browser using dev-browser skill
- Enhanced user profile tests pass: npx playwright test ui-users-enhanced.spec.ts
