# US-018: Render time series chart with Recharts

**Priority:** 18
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the Trends page renders a responsive line chart with two series (Total Events and Unique Users), tooltips on hover, a legend, and auto-scaling Y-axis. This is the core visualization for trend analysis.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- Install recharts in the client workspace
- Trends page renders a responsive line chart using Recharts (ResponsiveContainer + LineChart)
- Chart X-axis shows dates from the API response, formatted readably (e.g., 'Jan 5' for daily, 'Jan 5 - Jan 11' for weekly)
- Chart has two lines: one for total_count (labeled 'Total Events') and one for unique_users (labeled 'Unique Users')
- Lines use distinct colors that are accessible (e.g., blue for total, green for unique)
- Chart includes a Tooltip that shows the exact values on hover: date, total events count, unique users count
- Chart includes a Legend identifying both lines
- Chart Y-axis auto-scales based on data and starts at 0
- When the API returns all zeroes, chart still renders with the zero line visible and a subtle message: 'No events found in this range'
- Chart is responsive and fills the available width
- Typecheck passes
- Verify in browser using dev-browser skill
- Trends page chart tests pass: npx playwright test ui-trends.spec.ts (chart rendering, tooltip, legend tests)
