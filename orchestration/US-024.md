# US-024: Extend trend API with dimensional breakdown

**Priority:** 24
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the trend API supports breaking down any measure by a property value, returning multiple series (top 5 + __other__). Analysts can see revenue by plan type, events by URL, etc.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- GET /api/trends now accepts an additional optional parameter: breakdown_by (string, a property key)
- When breakdown_by is provided, instead of a single data array, the response returns: { ..., series: Array<{ key: string, data: Array<{ date: string, value: number }> }> }
- Each entry in series represents one distinct value of the breakdown property
- The series are limited to the top 5 values by total volume (sum of the measure across all time buckets), and all remaining values are grouped into an '__other__' series
- Breakdown works with all measures: total_count, unique_users, sum, avg, min, max
- When breakdown_by is not provided, the response uses the existing flat data array format (backward compatible)
- Zero-fill applies to each series independently (all series have entries for all time buckets)
- Typecheck passes
- Trend API breakdown tests pass: npx playwright test api-trends.spec.ts (breakdown_by tests)

## Spec anchors (READ these sections if the briefing is unclear)

- Edge cases worth adversarial testing:

  - Exactly 5 distinct values — no __other__ series
  - 6+ distinct values — __other__ series present
  - breakdown_by on a property that doesn't exist — empty series or error

## Risk notes

The ranking and __other__ grouping must happen in TypeScript, not SQL. SQL should GROUP BY time_bucket and property_value; application code determines top 5 and aggregates the rest into __other__. Getting this wrong produces incorrect charts.
