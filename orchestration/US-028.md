# US-028: Add chart type switcher for comparative visualization

**Priority:** 28
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, analysts can switch between line, bar, and area charts on the Trends page. Switching chart type is client-only (no re-fetch).

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- Trends page adds a chart type toggle in the control bar with three options represented by lucide-react icons: LineChart (line), BarChart3 (bar), AreaChart (area)
- Line chart: renders as Recharts LineChart (existing default behavior)
- Bar chart: renders as Recharts BarChart with the same data, colors, and tooltip
- Area chart: renders as Recharts AreaChart with semi-transparent fills under each line
- Switching chart type does NOT re-fetch data — it only changes the rendering
- All three chart types work with both single-series and breakdown (multi-series) views
- Default chart type is 'line'
- Selected chart type is visually highlighted in the toggle
- Typecheck passes
- Verify in browser using dev-browser skill
- Chart type switching tests pass: npx playwright test ui-trends.spec.ts (bar, area, line chart toggle tests)
