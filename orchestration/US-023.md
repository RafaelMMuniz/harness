# US-023: Extend trend API with numeric aggregation measures

**Priority:** 23
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the trend API supports sum, avg, min, and max of numeric event properties in addition to total_count and unique_users. Analysts can measure revenue, average order value, etc. over time.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- GET /api/trends now accepts an additional optional query parameter: measure (string, one of 'total_count' | 'unique_users' | 'sum' | 'avg' | 'min' | 'max', default 'total_count')
- When measure is 'sum', 'avg', 'min', or 'max', a second required parameter is needed: property (string, the property key to aggregate)
- Validated with Zod: if measure is a numeric aggregation, property must be non-empty; if measure is 'total_count' or 'unique_users', property is ignored
- Returns 400 if property is specified but contains non-numeric values for the given event type
- Response data array entries become: { date: string, value: number } where value is the result of the selected measure for that time bucket
- For 'sum': sum of the property values across all events in the bucket
- For 'avg': average of the property values across all events in the bucket
- For 'min': minimum property value in the bucket
- For 'max': maximum property value in the bucket
- Buckets with no events return value: 0 for sum, and value: null for avg/min/max (to distinguish 'no data' from 'zero')
- Existing total_count and unique_users measures continue to work as before (backward compatible)
- Typecheck passes
- Trend API numeric measure tests pass: npx playwright test api-trends.spec.ts (sum, avg, min, max tests)

## Spec anchors (READ these sections if the briefing is unclear)

- prd.json notes — "Use json_extract(properties, '$.' || property_name) in SQLite. Cast to REAL for aggregation. Handle NULL and missing property gracefully."

## Risk notes

SQLite's `json_extract(properties, '$.' || property_name)` returns TEXT. Must CAST to REAL for numeric aggregation. Events where the property is missing or null should be excluded from avg/min/max calculations (not treated as 0).
