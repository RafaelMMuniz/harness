# US-035: Add multi-event comparison to trends page

**Priority:** 35
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, analysts can select multiple events (up to 5) and plot them on the same chart for side-by-side comparison. Multi-event and breakdown are mutually exclusive. This is a Tier 4 nice-to-have (BR-402).

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- Trends page event selector allows choosing multiple events (multi-select with checkboxes or tags)
- When multiple events are selected and no breakdown is active, each event becomes a separate series on the chart with a distinct color and legend entry
- The measure applies to each event independently (e.g., 'Total Count' shows count per event, 'Sum of amount' shows sum per event — only events that have the selected property are included)
- Maximum 5 events can be compared simultaneously (disable adding more after 5)
- Multi-event comparison and breakdown_by are mutually exclusive — selecting a breakdown clears multi-event, and vice versa, with a brief informational message
- GET /api/trends is called once per selected event (parallel requests) and results are merged client-side
- Legend clearly labels each series with the event name
- Typecheck passes
- Verify in browser using dev-browser skill
- Multi-event comparison tests pass: npx playwright test ui-trends.spec.ts (multi-event tests)
