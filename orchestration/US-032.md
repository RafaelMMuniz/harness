# US-032: Implement global visual coherence pass

**Priority:** 32
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, MiniPanel looks and feels like a polished product: consistent color palette, typography hierarchy, loading/empty/error states on every page, readable chart labels, and smooth sidebar interactions.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- Define a consistent color palette in tailwind.config.ts: primary color (for actions and active states), chart colors (at least 6 distinct colors for multi-series charts), status colors (success green, warning yellow, error red), neutral grays for text hierarchy
- Typography hierarchy is consistent: page titles use the same size/weight, section headings use the same size/weight, body text and labels are consistent across all pages
- All pages have proper loading states: skeleton loaders or spinners while data is being fetched (Events, Trends, Funnels, User Profile)
- All pages have proper empty states with helpful messages: Events page ('No events yet. Run npm run seed to load sample data.'), Trends page ('Select an event to see its trend'), Funnels page ('Define at least 2 steps to analyze a funnel'), User profile ('Enter a user ID or device ID to look up their activity')
- All API error responses are caught and displayed as toast notifications or inline error messages (not raw error dumps or blank pages)
- Chart labels are readable: axis labels have appropriate font size, dates are formatted consistently, numbers use locale formatting (commas for thousands)
- No page shows a blank white screen or broken layout when data is missing or an API call fails
- Sidebar active state, hover states, and transitions are smooth and consistent
- Typecheck passes
- Verify in browser using dev-browser skill
