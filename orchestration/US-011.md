# US-011: Add distinct event names and basic stats API endpoints

**Priority:** 11
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the frontend can populate filter dropdowns with distinct event names and display summary stats (total events, total resolved users, event counts by name, date range).

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- GET /api/events/names returns an array of distinct event names sorted alphabetically
- GET /api/stats/overview returns a JSON object with: total_events (number), total_users (number of distinct resolved user identities), event_counts_by_name (object mapping event name to count), date_range (object with earliest and latest event timestamps)
- Total users count uses resolved identities: anonymous devices mapped to a user are not counted separately
- Responses are fast (< 500ms) on the sample dataset of 10k events
- Typecheck passes
- Query API tests pass (names + stats): npx playwright test api-queries.spec.ts

## Risk notes

The total_users calculation MUST use identity resolution. The SQL should count distinct resolved identities: `COALESCE(im.user_id, e.user_id, e.device_id)` where `im` is a LEFT JOIN on identity_mappings. A simple `COUNT(DISTINCT user_id)` is wrong — it misses anonymous devices and double-counts mapped devices.
