# US-034: Add autocomplete and search to event and property selectors

**Priority:** 34
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, all event name and property dropdowns across the app support type-to-search filtering, making it easy to find the right option in long lists. This is a Tier 4 nice-to-have (BR-401).

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- All event name dropdowns across the app (Trends, Funnels, Events filter) support type-to-search filtering
- All property dropdowns (measure property, breakdown property) support type-to-search filtering
- Use shadcn/ui Combobox (Command + Popover pattern) or a searchable Select component
- Searching is case-insensitive and matches anywhere in the name (not just prefix)
- When the list is long (>10 items), the input auto-focuses for immediate typing
- When the list is short (<=10 items), all items are visible without scrolling
- No change to API endpoints — filtering is client-side on already-fetched data
- Typecheck passes
- Verify in browser using dev-browser skill
