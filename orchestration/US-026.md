# US-026: Update trends page with measure and property selectors

**Priority:** 26
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the Trends page has a Measure dropdown (Total Count, Unique Users, Sum, Average, Min, Max) and a conditional Property dropdown for numeric aggregations. Analysts can analyze numeric properties like revenue over time.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- Trends page control bar adds a 'Measure' dropdown with options: Total Count, Unique Users, Sum, Average, Min, Max
- When Sum, Average, Min, or Max is selected, a 'Property' dropdown appears next to it, populated from GET /api/events/:eventName/properties filtered to only numeric properties
- When Total Count or Unique Users is selected, the Property dropdown is hidden
- Changing the measure or property re-fetches trend data with the new parameters
- The chart Y-axis label updates to reflect the measure (e.g., 'Total Events', 'Unique Users', 'Sum of amount')
- If no numeric properties exist for the selected event, Sum/Avg/Min/Max options are disabled in the dropdown with a tooltip: 'No numeric properties available'
- Default measure remains 'Total Count'
- Typecheck passes
- Verify in browser using dev-browser skill
- Trends page measure selector tests pass: npx playwright test ui-trends.spec.ts (measure/property selector tests)
