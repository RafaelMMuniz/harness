# US-036: Write automated tests for numeric aggregations and funnels

**Priority:** 36
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the codebase has server-side tests verifying that numeric aggregations (sum, avg) compute correctly and funnel analysis handles identity resolution across steps and enforces temporal ordering.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- Test file at server/src/__tests__/aggregations.test.ts
- Test 1 - Sum aggregation: insert 'Purchase Completed' events with known amount values across 3 days, call trends API with measure=sum and property=amount, assert each day's value matches the expected sum
- Test 2 - Average aggregation: same events as test 1, call with measure=avg, assert each day's value matches the expected average
- Test 3 - Non-numeric property rejected: call trends API with measure=sum on a string property, assert 400 response
- Test file at server/src/__tests__/funnels.test.ts
- Test 4 - Basic funnel: insert events for 10 users across 3 steps (all 10 do step 1, 7 do step 2, 3 do step 3), call funnel API, assert counts are 10, 7, 3 and conversion rates are 1.0, 0.7, 0.3
- Test 5 - Funnel with identity resolution: user does step 1 as anonymous device-X, then identifies as user-Y, then does step 2 as user-Y — assert the funnel counts this as 1 user who completed both steps
- Test 6 - Step order enforcement: user does step 2 before step 1 (by timestamp) — assert this user is NOT counted as completing step 2 in the funnel
- All tests pass
- Typecheck passes
