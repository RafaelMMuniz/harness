# US-027: Add breakdown selector to trends page

**Priority:** 27
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the Trends page has a "Break down by" dropdown that splits the chart into multiple color-coded series (one per property value), with __other__ in gray and a legend identifying each series.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- Trends page adds a 'Break down by' dropdown in the control bar, populated from GET /api/events/:eventName/properties (all types, not just numeric)
- Dropdown includes a 'None' option (default) that shows the single-series view
- When a breakdown property is selected, the API request includes breakdown_by parameter
- The chart renders multiple lines (one per series key) with distinct colors from a predefined palette of at least 6 colors
- A legend below or beside the chart shows the series key and its color
- The '__other__' series is rendered in gray and listed last in the legend
- Tooltip shows values for all visible series at the hovered date
- Changing the breakdown re-fetches and re-renders the chart
- Typecheck passes
- Verify in browser using dev-browser skill
- Trends page breakdown tests pass: npx playwright test ui-trends.spec.ts (breakdown selector, multi-series tests)

## Spec anchors (READ these sections if the briefing is unclear)

- prd.json notes — "When breakdown is active, response format changes from flat data[] to series[]. Use Recharts' multi-Line approach: map over series and render a <Line> for each."
