# US-017: Build trends page layout with controls

**Priority:** 17
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, MiniPanel has a Trends page accessible from the sidebar with event selector, date range presets, granularity toggle, and API wiring. Selecting an event fetches trend data. The chart rendering is deferred to US-018.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- New 'Trends' route added to the router, accessible from the sidebar navigation with a TrendingUp (or BarChart3) lucide-react icon
- Page heading: 'Trends'
- Control bar at the top contains: event name selector (dropdown populated from GET /api/events/names), date range preset selector (dropdown with 'Last 7 days', 'Last 30 days', 'Last 90 days', 'Custom'), custom date inputs (start and end date pickers, visible only when 'Custom' is selected), granularity toggle ('Day' | 'Week' buttons or segmented control)
- Default state on load: no event selected, date range 'Last 30 days', granularity 'Day'
- When no event is selected, the chart area shows a prompt: 'Select an event to see its trend'
- Selecting an event triggers an API call to GET /api/trends with the current control values
- Changing any control (event, date range, granularity) re-fetches data automatically
- Loading spinner or skeleton shown while data is being fetched
- Uses shadcn/ui Select, Button, and Input components
- Typecheck passes
- Verify in browser using dev-browser skill
- Trends page basic UI tests pass: npx playwright test ui-trends.spec.ts (navigation, default state, event selection, chart rendering)
