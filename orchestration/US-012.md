# US-012: Add user lookup page for individual user activity

**Priority:** 12
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, support leads can look up any user by user_id or device_id and see their complete resolved event timeline, associated device IDs, and profile summary (first/last seen, event count).

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- Users page is accessible from the sidebar navigation
- Shows a search input where the user can enter a user_id or device_id
- On search, calls GET /api/events?user_id=<id> (or device_id) and displays the resolved user's complete event timeline
- Timeline displays events in chronological order (oldest first) with: timestamp, event name, all properties expanded
- Shows the resolved identity at the top: 'User: <resolved_id>' with a list of associated device IDs from identity_mappings
- GET /api/users/:id endpoint returns user profile data: resolved user_id, list of mapped device_ids, total event count, first_seen and last_seen timestamps
- Empty state when no user/device is found with that ID
- Uses shadcn/ui components (Input, Card, Table)
- Typecheck passes
- Verify in browser using dev-browser skill
- User lookup tests pass: npx playwright test ui-users.spec.ts and npx playwright test api-users.spec.ts

## Test scope (for the validator)

- Primary test file: `e2e/US-012.spec.ts` or run `e2e/ui-users.spec.ts` and `e2e/api-users.spec.ts`
- Coverage the test MUST have (derived from acceptance criteria above):
  - GET /api/users/:id with known user_id returns profile with device_ids, total_events, first/last_seen
  - GET /api/users/:id with device_id resolves to mapped user
  - GET /api/users/:id with unknown ID returns 404

  - Search by user_id displays resolved identity heading
  - Search result shows associated device IDs
  - Event timeline in chronological order (oldest first)
  - Empty state for unknown ID
- Edge cases worth adversarial testing:
  - Search by device_id that has a mapping resolves to user profile
  - User with multiple devices shows all device IDs
