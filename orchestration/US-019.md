# US-019: Verify event explorer covers BR-200 requirements

**Priority:** 19
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the event explorer page fully satisfies BR-200: reverse chronological order, resolved identity column with visual distinction between user/anonymous, filter by event name, pagination, and live event appearance on refresh.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- Events page shows events in reverse chronological order (newest first) — verify this is the default sort and fix if not
- Each event row displays: timestamp (human-readable format), event name, resolved identity (show user_id if resolved, device_id if anonymous, with a visual indicator for anonymous), all properties (visible on expand or in detail panel)
- Filtering by event name works: selecting an event name from the dropdown shows only matching events
- Pagination works: 'Previous' and 'Next' controls navigate through large datasets without loading all events at once
- Sending an event via the API and then refreshing the events page shows the new event at the top of the list
- Typecheck passes
- Verify in browser using dev-browser skill
