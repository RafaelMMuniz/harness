# US-016: Add date range preset helper and validation

**Priority:** 16
**Status:** passes:false (briefing generated during --fresh)

## What this story accomplishes

After this story lands, the client has a shared utility module with date range presets (Last 7/30/90 days, Custom) and a date formatting function for API calls. The Trends page and future features use consistent time ranges.

## Acceptance Criteria (VERBATIM from prd.json — DO NOT paraphrase)

- A shared utility module exists in the client (e.g., client/src/lib/date-presets.ts)
- Exports a DATE_PRESETS array with objects: { label: string, value: string, getDates: () => { start: Date, end: Date } }
- Presets include: 'Last 7 days', 'Last 30 days', 'Last 90 days', 'Custom'
- 'Custom' preset returns null dates indicating the user must pick manually
- Each preset calculates dates relative to the current date at call time
- Exports a formatDateForAPI(date: Date): string function that returns ISO 8601 date string (YYYY-MM-DD)
- Typecheck passes
